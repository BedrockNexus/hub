# Security Policy

## Reporting A Vulnerability

Do not open a public issue for a suspected vulnerability.

Use GitHub's private vulnerability reporting feature on the repository's
Security tab. Include the affected route or function, reproduction steps,
impact, and a minimal proof of concept where appropriate. Remove credentials,
access tokens, personal data, and production content from the report.

We will acknowledge reports when reviewed, investigate their impact, and
coordinate a fix and disclosure where needed. Please allow a reasonable period
for remediation before publishing details.

## Supported Code

Security fixes target the current default branch and the production deployment.
Old forks, historical commits, and unsupported self-hosted modifications are
not maintained by the Bedrock Nexus team.
